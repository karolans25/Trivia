1) ¿Qué es CSS?

Las hojas de estilo en cascada (mejor conocidas como CSS) es un lenguaje que se usa para explicar y describir cómo se deben presentar las cosas escritas por lenguajes de marcado (como HTML). Mucha información para explicar qué es CSS ¿cierto? En pocas palabras CSS se usa para visualizar las cosas escritas con lenguaje HTML, como imágenes, fuentes, fondos y más. Cuando se trata de preguntas básicas sobre CSS en una entrevista de trabajo, los empleadores buscan que las personas puedan explicar las cosas con sus propias palabras y no recitar oraciones como robot. ¡Por eso deberías quedarte con las definiciones más sencillas!

2) ¿Cuál es la diferencia entre HTML y CSS?

Te sorprendería saber cuántas personas no saben la diferencia entre lo qué es CSS y lo qué es HTML. Su diferencia es simple: el lenguaje HTML se usa para escribir y crear contenido en la página web, mientras que CSS se usa para la parte de diseño - lo que incluye el fondo, diseños, fuentes, imágenes, etc.

3) ¿Cómo puedes aplicar estilos CSS a una página web?

En total hay tres maneras para aplicar estilos usando CSS. El comando link es el método más usado - simplemente cargas otro archivo CSS (con atributos) en el que estás trabajando. Esto es muy conveniente porque estás reutilizando los mismos archivos una y otra vez. También puedes utilizar el atributo style si sólo quieres alterar un sólo elemento (utilizarías el atributo en la misma línea del elemento), o puedes usar style al principio de tu archivo HTML, donde se localizan head.

4) ¿Cuántos tipos de CSS existen?

Ahora que sabes qué es CSS, debes saber que actualmente hay cinco tipos de CSS: CSS 1, CSS 2, CSS 2.1, CSS 3 y CSS 4. Seguramente te harán muchas preguntas sobre CSS3, porque es una de las versiones anteriores más comunes.

5) ¿Qué es un ‘framework’ CSS?

En CSS, los frameworks son simplemente bibliotecas de archivos prefabricados que pueden utilizarse para diseñar sitios web que requieren algunos diseños CSS estándar.

6) ¿Qué es la ‘focalización de objetos’?

Este es un buen ejemplo de algunas preguntas de CSS que recibirás en tu entrevista laboral, al menos al principio, junto qué es CSS, etc. El que te pidan que expliques ciertos elementos en específico de un programa o lenguaje de programación es una práctica común entre empleadores en todo el mundo. La focalización de elementos es cuando especificas ciertos elementos que quieres estilizar o alterar de alguna manera. En total, lo puedes hacer de tres maneras, por etiquetado, clase o por el ID del elemento que has especificado en el código HTML.

7) ¿Cómo especificarías los colores CSS?

Hay dos maneras para especificar colores CSS - la RGB y la del código de color hexadecimal. RGB viene del inglés red, green and blue. Muchas combinaciones distintas de estos colores (dependiendo del porcentaje que cada uno usa) pueden crear muchos tonos diferentes. Un código de color hexadecimal es un código que representa un color específico. Estos códigos se componen de seis letras y números diferentes (cada combinación representaría uno de los colores CSS).

8) ¿Qué son los ‘pseudo elementos’?

Si un párrafo de texto estaba en tu elemento, pero sólo quieres estilizar la primer letra de ese párrafo, tendrías que usar un pseudo elemento para esta tarea. Estos elementos se usan escribiendo dos puntos dobles frente la parte del elemento principal que quieres estilizar.

9) ¿Qué son las ‘pseudo clases’?

Las pseudo clases trabajan casi como los pseudo elementos, pero la diferencia principal es que alteran la apariencia de un elemento cuando está en un estado específico. ¿Te has dado cuenta cuando mueves tu cursor sobre un botón en la pantalla y el botón se resalta o estiliza de otra manera? Ese es un ejemplo de un elemento alterado por una pseudo clase.

10) ¿Cómo puedes integrar CSS a un sitio web?

Ahora, más allá de qué es CSS, vamos a otro ejemplo de una de las preguntas frecuentes en una entrevista de trabajo. Existen tres maneras para lograr esta tarea - las integraciones en línea, integrado e importado. El método en línea se usa para insertar una hoja de estilo CSS a un documento HTML. El integrado te permite insertar estilos únicos a un sólo documento, mientras que el importado te permite cambiar múltiples documentos.

11) Explica el concepto de ‘especificidad’

Esta es una de las preguntas en una entrevista CSS más complicadas porque seguramente tus empleadores no solamente quieren que expliques el concepto, pero también los tipos de especificidad en CSS. El concepto significa que ciertas reglas CSS reemplazan (sobre-escriben) a otras al ser más específicas. Este es un concepto complicado porque podría causar algunos problemas en el futuro - si usas muchos tipos de reglas de alta especificidad podría dificultar lo que necesitas cambiar una vez terminado el trabajo. Existen tres niveles de especificidad - tipo, clase y ID. Entre los selectores CS, el de tipo es la especificidad más baja mientras que ID es la más alta.

12) Explica las fuentes ‘seguras para web’ y ‘alternativas’

Cuando intentas elegir fuentes para tu sitio web, debes tener en cuenta que no todos los navegadores reconocen todos los tipos de fuentes - aquí es donde son útiles las fuentes seguras para web. Estas fuentes son populares y usadas comúnmente, como Times New Roman, Arial o Calibri. Si por alguna razón tu navegador no reconoce estas fuentes (aunque no debería suceder), hay fuentes alternativas - estas fuentes las elige tu navegador automáticamente. Los tipos de fuentes podría ser una de las preguntas frecuentes en una entrevista de trabajo CSS, asegúrate de memorizarlas.

13) ¿Qué es la ‘división de archivos’?

Esta es una de las preguntas en una entrevista CSS que se explican por sí mismas, la división de archivos te permite dividir archivos grandes en pequeños para que el programa pueda ejecutarse de manera rápida y eficaz. Para dividir archivos, necesitas un preprocesador CSS. Los archivos se pueden dividir como quieras, pero se recomienda pensarlo bien y mantenerlo ordenado. Esto te ayudará a gestionar más rápido tu sitio web sin tener que esperar a que carguen hojas de estilo adicionales.

15) Define un ‘conjunto de reglas’

En CSS, los conjuntos de reglas se crear con bloques de declaración que siguen a un conjunto específico de selectores CSS. Estos conjuntos de reglas básicamente le dicen al servidor cómo debe verse cierto documento.

16) Define ‘selector universal’ y da un ejemplo

Los selectores universales son los que te dejan recoger todos los archivos con el mismo nombre (sin importar el tipo de elemento), en lugar de recoger específicos.

17) ¿Qué recogen los selectores CSS ‘clase’ y ‘ID’ cuando se emiten?

Un selector de clase recogerá un bloque completo de información, mientras que el selector ID te permite recoger un elemento específico que posee un tipo único de número ID. Ya hablamos de clases y IDs antes, pero esta es una pregunta importante sobre CSS. Saber las diferencias y diferentes funciones entre clases, tipos y IDs podría decidir tu éxito en la entrevista laboral CSS.

18) ¿Qué es ‘float’ en CSS?

La propiedad float te permite mover imágenes a través de un muro de texto en tu documento HTML sin tener que partir el texto. Tal como en Google Docs, el texto envuelve a la imagen.

19) ¿Qué es el ‘índice Z’?

Un índice Z en CSS resalta cualquier o todas las partes que se traslapan entre tu estilo cSS y el documento en lenguaje HTML. Como la superposición es un problema común al estilizar y ajustar imágenes en CSS, es de hecho muy útil.

20) ¿Por qué usar import al principio del archivo?

import se utiliza comúnmente al principio de los archivos porque hay un gran miedo de sobreescribir conjuntos de reglas - que ya existen. Al colocar is import al principio, este problema se puede prevenir.
